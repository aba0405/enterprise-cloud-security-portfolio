# Challenge 01 – Amazon ECR Authentication Failed

## Phase

Phase 02 – Amazon ECR

---

## Scenario

After creating the Amazon ECR repository, I attempted to authenticate Docker with Amazon ECR using the AWS CLI.

Command:

```bash
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin 649966626634.dkr.ecr.us-east-1.amazonaws.com
```

---

## Error

```text
An error occurred (AccessDeniedException) when calling the GetAuthorizationToken operation:

User:
arn:aws:iam::649966626634:user/AWS_Practice

is not authorized to perform:

ecr:GetAuthorizationToken

because no identity-based policy allows the ecr:GetAuthorizationToken action.
```

Docker also returned:

```text
error: cannot perform an interactive login from a non TTY device
```

---

## Root Cause Analysis

The issue was **not related to Docker or Amazon ECR**.

The AWS CLI attempted to request a temporary authentication token from Amazon ECR, but the IAM user (`AWS_Practice`) lacked the required permission:

```text
ecr:GetAuthorizationToken
```

Since AWS denied the request, Docker received an empty authentication token and displayed a secondary login error.

---

## Solution

Granted the IAM user the AWS managed policy:

```text
AmazonEC2ContainerRegistryPowerUser
```

Waited approximately one minute for IAM permission propagation and reran the authentication command.

Result:

```text
Login Succeeded
```

---

## Validation

Verified Docker was successfully authenticated with Amazon ECR and proceeded to tag and push the container image.

---

## Lessons Learned

- Amazon ECR authentication depends on IAM permissions.
- Always read AWS error messages from top to bottom.
- Identify:
  - **Who** is making the request.
  - **What** action is being performed.
  - **Why** access was denied.
- The Docker error was only a symptom of the IAM authorization failure.

---

## Interview Takeaway

**Question**

Why did Docker fail to log in to Amazon ECR?

**Answer**

Docker authentication to Amazon ECR relies on an authorization token generated by AWS. My IAM user did not have permission to call the `ecr:GetAuthorizationToken` API, so AWS denied the request. After granting the required IAM permissions, Docker successfully authenticated to the private Amazon ECR registry.

---

## Architecture Principle

Authentication and authorization are separate concerns.

- Docker handles container operations.
- Amazon ECR stores container images.
- IAM determines whether the caller is authorized to access Amazon ECR.

Understanding which AWS service is responsible for each operation makes troubleshooting significantly easier.

# Decision 03 - Enable Amazon Inspector

## Decision

Use Amazon Inspector to automatically scan container images stored in Amazon ECR.

## Reason

Detect operating system and application package vulnerabilities before containers are deployed to Amazon ECS.

## Benefits

- Continuous vulnerability monitoring
- Automatic CVE detection
- Native AWS integration
- Reduced deployment risk
